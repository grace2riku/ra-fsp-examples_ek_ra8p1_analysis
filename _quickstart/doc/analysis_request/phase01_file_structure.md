# Phase 1: ファイル構造の把握

## カテゴリ
環境・構成編

## 概要
プロジェクト全体のディレクトリ構造を把握し、各ファイル・フォルダの役割を明確にする。

## 解析要求事項

このプロジェクト全体のディレクトリ構造を解析し、ファイルツリー形式で各ファイル・フォルダの役割を説明してください。

### 分類観点

以下の観点で分類・説明してください：

1. **アプリケーションコード**
   - ユーザーが編集するコード
   - メインの処理ロジック

2. **FSP自動生成コード（ra_gen/）**
   - FSPコンフィグレータが生成するコード
   - 編集すべきでないファイル

3. **FSP設定ファイル（ra_cfg/）**
   - 各ドライバの設定ヘッダ
   - ユーザー設定可能な項目

4. **FSPドライバ・ライブラリ（ra/）**
   - HALドライバのソースコード
   - CMSISファイル

5. **スタートアップ・リンカスクリプト**
   - 起動処理関連ファイル
   - メモリ配置定義

6. **プロジェクト設定ファイル（e2studio/）**
   - IDE設定ファイル
   - ビルド設定

7. **その他のリソース**
   - 画像データ
   - フォントデータ
   - その他のアセット

### 出力形式

- ツリー形式でディレクトリ構造を表示
- 各ファイル/フォルダに簡潔な説明を付記
- 重要なファイルは★マークで強調
- ファイル数が多い場合は代表的なものを記載し、省略部分は「...」で表記

### 出力例

```
quickstart_ek_ra8p1_ep/
├── e2studio/                          # ★ e2 studio プロジェクト設定
│   ├── configuration.xml              # ★ FSP設定ファイル（クロック、ピン、周辺機能）
│   ├── .project                       # Eclipseプロジェクト設定
│   └── .cproject                      # C/C++プロジェクト設定
├── src/                               # ★ アプリケーションソースコード
│   ├── hal_entry.c                    # HALエントリポイント
│   └── ...
...
```

## 期待する成果物

1. 完全なディレクトリツリー（説明付き）
2. 重要ファイルの一覧と役割説明
3. ファイル編集時の注意事項（自動生成 vs 手動編集）
