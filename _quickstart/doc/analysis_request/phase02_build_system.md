# Phase 2: ビルドシステム・環境構成

## カテゴリ
環境・構成編

## 概要
プロジェクトのビルドシステムと開発環境を解析し、ビルド手順と出力ファイルの場所を明確にする。

## 解析要求事項

このプロジェクトのビルドシステムと開発環境を解析してください。

### 1. ビルドシステムの構成

- 使用しているIDE（e2 studio）の設定ファイル
- Makefileまたはビルドスクリプトの有無
- FSP (Flexible Software Package) のバージョン
- ツールチェーン（GCC ARM, IAR, ARMCCなど）
- ツールチェーンのバージョン

### 2. プロジェクト設定ファイル

- `.project`, `.cproject` の役割と主要設定
- `configuration.xml` の構成
- ビルド構成（Debug/Release）の違い
- ビルド構成の切り替え方法

### 3. ビルド手順

- e2 studioでのビルド方法（GUI操作）
- コマンドラインでのビルド方法（可能な場合）
- FSPコード生成の手順
- ビルド時の依存関係
- クリーンビルドの方法

### 4. 出力ファイル

以下のファイルの生成場所とファイル名を特定してください：

| ファイル種別 | 拡張子 | 用途 |
|-------------|--------|------|
| 実行ファイル | .elf | デバッグ用 |
| HEXファイル | .hex | フラッシュ書き込み用 |
| MOTファイル | .mot/.srec | フラッシュ書き込み用 |
| MAPファイル | .map | メモリ配置確認用 |
| オブジェクトファイル | .o | 中間ファイル |
| 依存関係ファイル | .d | 依存関係追跡用 |

### 5. デバッグ設定

- デバッガの種類（J-Link, E2 Lite など）
- デバッグ接続設定（SWD/JTAG）
- デバッグ構成ファイル（.launch）の場所
- ブレークポイント、ウォッチ設定の保存場所
- デバッグ開始手順

### 6. デュアルコア構成の場合

- CM85とCM33それぞれのビルド設定
- マルチコアデバッグの設定
- コア間の依存関係

## 期待する成果物

1. **ビルドフロー図**
   - ソースファイルから実行ファイルまでの流れ
   - 各ステップで使用されるツール

2. **設定ファイル関係図**
   - 各設定ファイルの関係性
   - どのファイルがどの設定に影響するか

3. **クイックリファレンス**
   - よく使うビルドコマンド
   - トラブルシューティング情報
